KEY FINDINGS:

1. PasswordlessFlow POSTs directly to the MCP boundary (/auth/login-request),
   NOT to Keycloak directly. The MCP boundary resolves the realm on behalf
   of the caller.

2. The realm parameter defaults to "kh-prod" and is sent in the JSON body.

3. This path is kh-prod-aware by default and does not depend on
   DEFAULT_AUTH_SERVER.

4. PasswordlessFlow uses mcp_base_url, not auth_server_url. It is structurally
   isolated from the PKCE/device realm binding issue.

5. The auth split is isolated to PKCE and device flows only.
   Passwordless is already aligned with kh-prod intent.
