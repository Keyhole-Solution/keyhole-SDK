KEY FINDINGS:

1. PKCEFlow and DeviceFlow are constructed at __init__ time with auth_server_url.
   The URL containing the realm is baked into the flow object at construction.

2. realm parameter is accepted by login() but is NOT used to construct,
   reconfigure, or redirect PKCEFlow or DeviceFlow.

3. The only flow where realm reaches the network is PASSWORDLESS.

4. Effective issuer for PKCE: https://auth.keyholesolution.com/realms/keyhole-mcp
   Effective issuer for DEVICE: https://auth.keyholesolution.com/realms/keyhole-mcp
   Effective realm for PASSWORDLESS: kh-prod (sent to MCP boundary, not Keycloak directly)

5. Conclusion: realm parameter is flow-specific. Auth split is primarily
   PKCE/device issuer binding, not passwordless behavior.
