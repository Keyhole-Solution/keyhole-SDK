KEY FINDINGS:

1. CLI login surface INTENDS realm="kh-prod" in both the Typer option default
   and in the run_login() function default.

2. However, auth_server_url defaults to DEFAULT_AUTH_SERVER which embeds
   "keyhole-mcp" in its URL path.

3. The realm parameter is NOT used to rewrite auth_server_url at the CLI
   dispatch layer. It is passed as-is to AuthBootstrapClient.login().

4. For PKCE and device flows, AuthBootstrapClient uses auth_server_url
   directly (PKCEFlow and DeviceFlow constructors receive auth_server_url).
   The realm parameter is silently dropped for these flow paths.

5. Therefore the CLI intent (kh-prod) conflicts with the effective issuer
   (keyhole-mcp) for PKCE and device flows.

6. The realm parameter only reaches the network for the passwordless flow
   (POST /auth/login-request payload includes {"realm": realm}).
